# Incident Runbook

1. Detect and scope
- Identify impacted users, endpoints, and start time.

2. Stabilize
- Apply lowest-risk mitigation first (rate-limit, rollback, disable non-critical jobs).

3. Diagnose
- Gather logs, failing queries, and rule evaluation symptoms.

4. Recover
- Verify key transactions/auth flows are healthy.

5. Review
- Record root cause, customer impact, and prevention actions.
